# Package ownership

Prodivix 采用“Canonical documents + domain owners + revision-bound projections”。应用组合能力，稳定 contract 归对应 package；当前进度不在本文维护，见 [`specs/roadmap/current-status.md`](../../specs/roadmap/current-status.md)。

## 稳定依赖方向

```mermaid
flowchart TD
    Web["apps/web"] --> UI["UI / React projections"]
    Web --> Domain["Domain owners"]
    UI --> Domain
    Compiler["prodivix-compiler"] --> Domain
    Browser["runtime-browser"] --> Core["runtime-core"]
    Browser --> Domain
    Domain --> Behavior["behavior contribution contract"]
    Behavior --> Authoring["authoring / semantic-index"]
    Behavior --> Core
    Verification["verification"] --> Behavior
    Verification --> Authoring
    Verification --> Core
    Web --> Behavior
    Web --> Verification
    RemoteAdapters["Control Plane / Worker adapters"] --> RemoteCore["runtime-remote"]
    RemoteCore --> Core
    Backend["apps/backend"] --> Sync["Workspace operation contract"]
    Backend --> Verification
```

领域层不得依赖 React、DOM、具体 fetch transport 或编辑器内部 store。跨领域语义通过 Semantic Contribution/Query 协作，不互相扫描内部结构。

## Ownership 变更规则

1. 新稳定 contract 必须先确定 owner；adapter 与 composition root 不能以便利为由复制 contract。
2. Canonical Workspace document 仍由领域 owner 定义；Semantic Index、renderer、runtime、Git 与 compiler 只拥有 projection。
3. package owner 发生变更时，应同时更新相应 ADR、import boundary Gate 与本文件；不要在 `AGENTS.md` 追加临时表格。

## 背景

G2 已分别建立 Route、PIR event、Data dispatch、NodeGraph ExecutionProvider、Animation playback、
Auth/Server Runtime 与 Workspace Test。它们能够在 exact Workspace revision 上运行，但组合关系仍由
领域私有结构、少量硬编码 action 或产品 fixture 表达，尚未形成可寻址、可影响分析、可跨 target 重放的
行为契约。

G3 不能把一组 UI 点击脚本称为行为模型，也不能建立一个吞并 Route、NodeGraph、Animation 和 Data 的
巨型 Behavior document。产品需要一个很薄的跨领域协议：领域 owner 继续拥有实际语义，Scenario 只描述
可重复的外部行为、控制和断言，并通过稳定引用连接这些语义。

## 决策

新增目标 package `@prodivix/behavior`，它只拥有以下跨领域 contract：

1. `BehaviorScenario` current domain model 与 wire codec/migration。
2. `BehaviorTrigger`、`BehaviorAction`、`BehaviorObservation` 和 stable step/assertion identity。
3. Scenario normalize、validate、compile 与 semantic contribution。
4. transport-neutral `BehaviorScenarioProgram`，供 Preview、Export 和 CI adapter 消费。

它不拥有 Route matching、Data lifecycle、NodeGraph scheduling、Animation evaluation、DOM、Browser automation、
ExecutionProvider 或 VerificationEvidence。

### Canonical Workspace document

Scenario 使用新的 typed Workspace document `behavior-scenario`。它与 PIR、NodeGraph、Animation、DataSource
并列，由 `core.behavior` Command namespace 管理；创建、编辑、录制采纳、baseline 更新和删除均进入可逆
Command/Transaction、Durable Outbox 与 Atomic Commit。

领域 current model 不暴露数字版本。数字版本只存在于 immutable wire snapshot、codec、migration 与
Backend persistence validator。

概念结构如下，最终字段由 implementation contract 和 schema 冻结：

```ts
type BehaviorScenario = Readonly<{
  id: string;
  name: string;
  description?: string;
  criticality: 'smoke' | 'standard' | 'critical';
  tags: readonly string[];
  entry: BehaviorScenarioEntry;
  fixtureRefs: readonly BehaviorFixtureReference[];
  controls: BehaviorControlProfile;
  lanes: readonly BehaviorScenarioLane[];
  checkpoints: readonly BehaviorCheckpoint[];
  assertions: readonly BehaviorAssertion[];
  matrixHints?: BehaviorScenarioMatrixHints;
}>;
```

每个 lane、step、checkpoint 和 assertion 都有稳定 id。数组位置、显示名称、DOM selector、测试工具 task id
或录制序号不能充当长期 identity。

### Trigger 与 Action

`BehaviorTrigger` 只表达已经由 owner 注册的稳定事件：

- Route：enter、ready、leave、navigation rejected；
- PIR：document/node event；
- Data：query/mutation lifecycle、stream item、retry、optimistic settle；
- NodeGraph：graph invocation、node/output event；
- Animation：timeline lifecycle 与 named marker；
- Auth/Server：principal/session transition 与 sanitized invocation outcome；
- Test/Scenario：fixture ready、barrier、checkpoint 与 assertion observation。

`BehaviorAction` 使用 discriminated union 和 document-qualified reference，至少覆盖：

- navigate；
- dispatch DataOperation；
- invoke NodeGraph；
- play/pause/resume/seek/cancel Animation；
- update temporary application state；
- invoke registered CodeSlot；
- wait for typed observation 或 barrier。

每个 action descriptor 必须声明 input/output type、runtime zone、required capabilities、effect class、
cancellation semantics 与 SourceTrace resolver。Data mutation、Server Function 或其他副作用继续复用 G2 的
permission、idempotency、Secret 和 replay gate；Behavior 不包装第二套网络或 mutation API。

### Scenario 不是应用行为的第二真相源

实际应用行为仍由原领域保存：

- PIR event 通过 typed action reference 调用 NodeGraph/Data/Animation；
- Route lifecycle 由 Router contract 管理；
- control/data flow 由 NodeGraph document 管理；
- timeline 与 effect 由 Animation document 管理；
- Data lifecycle 与 Auth/Server policy 由其领域 document 管理。

Scenario 负责驱动入口、用户动作、故障控制和观察结果。它可以直接调用明确标记为 test/system action 的
能力，但不能复制一份生产 action graph 来“模拟”应用。Scenario 与真实行为漂移时，semantic diagnostics
必须阻断 verification，而不是让测试脚本继续跑绿。

### Target 与 observation

Scenario 禁止保存 CSS/XPath、DOM 序号、React component instance、编辑器 store path 或供应商 locator。
可持久化 target 只能是：

1. `DiagnosticTargetRef` 可表达的稳定领域目标；
2. Workspace Semantic Index 中的 symbol/reference；
3. Route、Component public contract、accessible role/name 等稳定公共语义；
4. 编译器生成的 opaque verification target id，并由 SourceTrace manifest 反解到作者态。

Observation 包括 route、visible/enabled/value、Data lifecycle、Network/Console absence、NodeGraph output、
Animation state、accessible tree、visual checkpoint 与自定义 pure CodeSlot assertion。自定义 assertion 只能通过
revision-bound CodeReference 进入 test zone，不能保存裸函数或任意 `eval` 字符串。

### Recorder 边界

Recorder 是 adapter，不是写入 owner：

```text
Browser interaction stream
  -> bounded recorder draft
  -> semantic target resolution
  -> unresolved/ambiguous review
  -> BehaviorScenario Command/Transaction
  -> Atomic Commit
```

原始 pointer/key event、DOM snapshot、selector 和页面文本只存在于有界 draft。无法解析到稳定 target 的事件
必须显式修复或丢弃，不能保存为“临时 selector”。密码、token、输入框敏感值和 Secret 不进入 draft、Scenario、
diagnostic 或 evidence。

### Semantic Index contribution

Behavior provider 为 Scenario、step、checkpoint、assertion 和它们引用的 Route/PIR/NodeGraph/Animation/Data/Code
目标提供 symbol/reference/dependency。这样 Workspace Semantic Index 可以完成：

- definition/reference navigation；
- scenario coverage reverse query；
- Change impact 扩展；
- orphan target、ambiguous target、type/capability mismatch diagnostics；
- refactor/rename planning。

Provider 只读取 exact Workspace snapshot；不得扫描 Web route state、DOM、测试报告或 Execution Session。

### Compile 与运行

Scenario compiler 把 exact Scenario revision、Workspace Semantic Index、Executable Project Snapshot 和 control
profile 编译为 immutable `BehaviorScenarioProgram`。Program 包含已解析 target、typed actions、assertions、
barrier、预算、SourceTrace 和 fixture digest，不包含 Workspace 全量镜像、credential 或 adapter object。

Preview、Export 和 CI 使用同一 Program contract。adapter 可以不同，但必须通过相同 conformance suite；
unsupported capability 形成 plan/diagnostic failure，不能静默跳过 step。

## Owner 边界

`@prodivix/behavior` 拥有跨领域引用与 Scenario contract，不执行领域内部语义。

依赖方向固定为领域 contribution/adapter 依赖 `@prodivix/behavior` 的公开 descriptor contract，再由应用、Compiler
或 Runtime composition root 聚合 registry；`@prodivix/behavior` 不反向 import Route、Data、NodeGraph、Animation 或
React/Vue package，避免领域环依赖。

各领域 owner 负责：

- action/trigger descriptor 与 validator；
- runtime adapter 和 SourceTrace；
- Command planner 与 domain migration；
- 不变量、错误语义和 capability declaration。

`@prodivix/workspace` 组合新 document type、Command domain、codec 与 semantic snapshot；`apps/backend` 执行
current wire validation 和 Atomic Commit；`apps/web` 只组合 editor/recorder/runner UI。

## 拒绝的方案

### 保存 Playwright/Cypress 脚本作为 canonical Scenario

拒绝。工具脚本持有 selector、等待和供应商 API，无法支持 Semantic Index、跨 target 或稳定 migration。

### 新建一个吞并所有行为的巨型 BehaviorGraph

拒绝。NodeGraph、Animation、Route 和 Data 已有明确 owner；巨型 graph 会复制领域语义并形成第二运行时。

### Recorder 直接写 Workspace

拒绝。浏览器事件噪声、敏感输入和 selector 不满足可逆写入与 semantic validation。

### Scenario 直接读取编辑器 React state

拒绝。Scenario 必须能在 Export 与 CI 运行，编辑器私有状态不是产品行为输入。

## 后果

- Workspace 与 Backend 需要新增 `behavior-scenario` typed document、`behavior` Command domain 和 validator。
- Semantic Index 增加 Behavior contribution provider 与 coverage query projection。
- Compiler/Runtime 增加 Scenario Program 和 verification target manifest，但不增加生产写路径。
- 录制、导入和 baseline 采纳必须经过正式 Command/Transaction。
- G4 Agent 后续只能创建/修改 Scenario proposal，不能绕过本 contract。

## 验收

- [ ] Scenario current/wire/codec/migration 与前后端 validator conformance 完成。
- [ ] stable step/assertion id、typed target/action/trigger 与硬预算完成。
- [ ] Scenario 创建、编辑、录制采纳、undo/redo、save/reload 走唯一 Workspace 写入链。
- [ ] Behavior semantic contribution 支持 definition/reference/impact/orphan diagnostics。
- [ ] Preview、Export、CI 消费同一 Scenario Program，adapter 私有 selector 不泄漏。
- [ ] Scenario 不复制应用行为、不保存 credential，也不直接写 runtime/Workspace state。

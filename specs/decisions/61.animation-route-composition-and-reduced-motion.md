## 背景

G2 Animation 已完成单 timeline evaluator、same-context provider、RAF/effect port、generation lease和Editor Session；
Route 已有guard/loader/action与layout/outlet composition。仍缺少Blueprint event到Animation的typed action、multi-timeline
composition、route enter/leave transition、pause/resume/seek、reduced-motion和跨target visual evidence。

若这些语义由Web组件的`useEffect`、CSS class或路由动画库私有配置实现，Preview、Export与CI会再次形成多套生命周期，
旧navigation/effect也可能覆盖新revision。

## 决策

### Typed Animation action

`@prodivix/animation`向Behavior registry贡献typed action/observation：

- `animation.play`：document/timeline、target scope、composition slot与play options；
- `animation.pause/resume/seek/cancel`：绑定instance/owner identity；
- `animation.marker`、started/completed/cancelled/failed observation；
- route transition request/result。

引用必须document-qualified并由Semantic Index解析。PIR event、NodeGraph node和Route lifecycle只保存typed reference；
禁止裸timeline id、CSS selector、DOM element或Web store callback。

### Composition model

Animation current model扩展显式composition：

- sequence、parallel、stagger和nested timeline；
- named marker/checkpoint；
- target scope与parameter binding；
- per-contributor priority、blend/replace policy；
- cancellation/error propagation；
- finite/infinite child约束与total duration推导。

同一target/property的并发贡献必须由domain planner在运行前解决：

- 可组合数值/transform使用明确blend order；
- replace使用stable priority和owner identity；
- 不兼容filter/string/shader冲突fail closed；
- 完成/取消cleanup只撤销本instance contributor，不覆盖其他owner或新generation。

不允许依赖React render顺序、RAF完成时刻或CSS cascade偶然决定winner。

### Route lifecycle

Router current contract增加可选typed transition bindings：enter、ready、leave和error。navigation lifecycle：

1. resolve/guard/loader；
2. prepare outgoing/incoming target scope；
3. run leave/enter composition under one navigation generation；
4. commit route visibility/outlet handoff at declared marker；
5. complete or cancel cleanup。

新navigation、guard rejection、loader error、timeout或target disappearance取消旧transition。generation lease保证旧frame/marker
不能提交新route状态。Route仍拥有navigation truth；Animation不能自行改变RouteManifest或URL。

SSR/static build不执行动画；client hydration后按route runtime contract启动。Preview、React/Vite、Vue/Vite与Remote Preview client
使用同一compiled composition语义。

### Reduced-motion policy

每个timeline/composition声明motion intent：

- `decorative`：reduced模式默认禁用或跳到最终stable frame；
- `spatial`：保留必要状态关系但缩短/替换大位移；
- `essential`：保留功能反馈，必须提供bounded reduced variant；
- `continuous`：reduced模式停止或使用static representation。

author可以提供reduced variant/ref；没有合法fallback时由canonical policy生成安全策略并发布warning。简单把所有duration设为0
不是通用方案，因为会跳过marker、route handoff、cleanup或最终状态。

system preference、project policy和verification override按稳定优先级解析；runtime只能收到resolved policy，不能各adapter自行猜。
Verification Plan对受影响animation/route强制full/reduced两个matrix cell，并断言功能结果一致、禁止动画满足policy。

### Pause、resume 与 seek

Playback基于explicit clock和instance state支持pause/resume/seek：

- pause停止scheduler但保留instance-ownedframe/effect lease；
- resume从logical cursor继续，不累计wall-clock pause；
- seek通过deterministic evaluator计算frame和marker crossing policy；
- route transition默认不可任意seek，debug/verification profile可受控seek；
- infinite timeline必须有cancel和verification checkpoint/freeze policy。

Scrub与runtime playback仍分离；editor authoring cursor不偷偷控制active production session。

### CodeSlot 与 shader

custom easing、timeline script和shader只通过revision-bound CodeSlot：

- easing必须pure、deterministic、bounded并返回finite scalar；
- timeline script只能生成声明允许的parameter/marker，不直接操作DOM/network/Workspace；
- shader声明GPU/runtime capability、input/output和fallback；unsupported target fail closed或使用显式author fallback；
- verification required cell不能在缺少capability时静默跳过。

GPU pixel variance按visual baseline compatibility key管理。G3 first vertical不要求离线视频、远程逐帧RPC或任意WebGPU compute。

### Visual/Accessibility observation

Animation observation只发布bounded timeline/marker/target/property summary和SourceTrace，不记录每帧Job event。Visual artifact在
Scenario checkpoint由verification browser adapter capture；performance adapter记录frame/jank aggregate，不保存raw high-frequency trace。

Reduced motion必须同时验证：

- route/navigation与业务结果不丢失；
- focus、accessible name/state和reading order不因transition失效；
- decorative/continuous motion遵守policy；
- cleanup后无旧effect、listener、scheduler或hidden inert target泄漏。

## Owner边界

`@prodivix/animation`拥有composition/evaluator/conflict/reduced policy/runtime port；`@prodivix/router`拥有navigation lifecycle和
transition binding；`@prodivix/runtime-browser`只投影frame/effect；Compiler生成framework target adapter；`@prodivix/behavior`
只连接typed action/observation；Web只组合产品表面。

## 拒绝的方案

### Route组件私有CSS class切换

拒绝。它绕过domain lifecycle、generation、reduced policy和SourceTrace。

### Reduced motion统一duration=0

拒绝。它可能跳过marker、handoff和cleanup，也无法表达essential motion fallback。

### Remote逐帧传输

拒绝。Remote只交付snapshot/bundle；动画在client runtime执行。

### visual screenshot就是Animation Evidence全部

拒绝。还必须验证lifecycle、functional outcome、reduced policy、a11y和cleanup。

## 后果

- Animation/Route current model、wire migration、Backend validator、Semantic Index和Compiler扩展。
- Browser effect host增加composition contributor和pause/resume/seek generation semantics。
- React/Vue target共享runtime contract，不各自实现transition DSL。
- Scenario/Verification能对marker、motion axis、visual/a11y/performance建立Evidence。

## 验收

- [ ] typed play/control/marker和Route lifecycle binding通过前后端codec/validator。
- [ ] sequence/parallel/stagger/nested/conflict/cancel在explicit clock下确定。
- [ ] route handoff、replacement navigation和late frame使用generation fence。
- [ ] full/reduced mode功能等价、a11y与cleanup Gate通过。
- [ ] CodeSlot/shader capability、determinism、fallback和target matrix fail closed。
- [ ] React/Vue Preview/Export/CI invoked composition通过同一Scenario。

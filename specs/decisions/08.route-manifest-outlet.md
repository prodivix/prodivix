## 决策摘要

Workspace RouteManifest 是项目路由的 canonical contract。`@prodivix/router` 持有 codec、normalizer、matching、navigation、module composition 与 validator；`@prodivix/workspace` 持有 Route planner 到 Command / Transaction 的集成。

Preview、导航 action、PIR Outlet、后端校验、Export topology 与 Diagnostics 消费同一 composed RouteGraph。

## Route identity 与文档引用

1. Route node id 是稳定 identity，path segment 是可编辑路由属性。
2. `pageDocId`、`layoutDocId` 与 outlet binding 使用 Workspace document id。
3. Route path 与 Workspace VFS path 属于不同语义。
4. route create、move、rename、bind、unbind 与 delete 通过可逆 Command 或原子 Transaction。
5. RouteManifest 修改推进 `routeRev`；涉及 Workspace tree/document metadata 时同时声明相应 revision preconditions。

## Outlet composition

```text
Workspace RouteManifest
  -> compose route modules
  -> match concrete preview/runtime path
  -> RouteRuntimeContext.matchChain
  -> resolve page/layout documents
  -> project default/named outlet content
  -> PIR React Renderer / Export Program
```

规则：

1. Active route 由稳定 route id 与 concrete path 共同描述。
2. `RouteRuntimeContext` 承载 match chain、params、search params、hash、page/layout refs 与 outlet binding。
3. `PdxOutlet` 是 RouteRuntimeContext 的 PIR projection，不建立独立路由系统。
4. Component route module 使用相对 RouteGraph，并由 host RouteManifest 显式 mount。
5. module composition 保留 source trace，可回到 host route、module route 与源文档。

## Runtime code slots

loader、action 与 guard 使用 CodeReference / CodeSlotContract：

1. Route document 保存稳定 CodeReference。
2. Code Authoring Environment 提供编辑、Language Service 与 SourceSpan。
3. Workspace Semantic Index 连接 route、param、code symbol、definition/reference 与 impact query。
4. Runtime execution 通过 Route Runtime adapter / ExecutionProvider 接入。

## Validation 与 diagnostics

Route validator 至少检查：

1. root、route id、parent/children、sibling segment/index 与 cycle。
2. page/layout document reference 与 document type。
3. module mount identity、relative path 与 dependency cycle。
4. default/named outlet binding 与 target compatibility。
5. loader/action/guard CodeReference shape 与 capability。

诊断使用稳定 route id、document id、module id 与 SourceTrace，并进入 `@prodivix/diagnostics` provider snapshot。

## Preview 与 Export

当前能力：

1. 地址栏、navigate action 与 Canvas preview 使用同一 route matcher/resolver。
2. PIR React Renderer 消费 RouteRuntimeContext 并投影 active page/outlet。
3. 后端 bootstrap 与 Atomic Commit 使用 route semantic validator。
4. ExportProgram 包含 route topology、runtime refs、document mappings 与 SourceTrace。
5. `.prodivix/routes.json` 与 export manifest 保存同一 topology summary。

G1 交付：

1. React/Vite target 完成 layout chain 与 route-outlet component composition。
2. 独立导出项目验证 route topology、navigation、params、layout/outlet 与 runtime adapter。
3. Component Instance、Collection 与 Route composition 共享 document identity、Semantic Index 与 SourceTrace。

## 长期不变量

1. RouteManifest 是项目路由唯一保存态。
2. Router、Renderer、Compiler、Backend 与 AI 使用同一 route contract。
3. Route writes 使用 Command / Transaction、WorkspaceOperation、Durable Outbox 与 Atomic Commit。
4. Route code-owned 能力使用 CodeReference / CodeSlot。
5. Route 与 document 的跨域关系进入 Workspace Semantic Index，同时保持类型化持久化。

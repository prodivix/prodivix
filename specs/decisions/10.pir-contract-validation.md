## 决策摘要

Prodivix 只维护一套无版本号的 `PIR-current` 领域模型。Canonical Workspace、
Workspace Semantic Index、Renderer、Compiler 与作者界面全部消费该模型，不消费数字
版本类型，也不按版本号分派行为。

`PIR-current.version.json` 是 activation manifest，指向当前不可变
`PIR-v<version>.json` wire snapshot。同步流程由该 snapshot 生成
`specs/pir/PIR-current.json`、generated `PIRWire*` 与后端 current schema/constants。
这些派生产物只服务 codec/persistence 边界；wire document 必须先经过 strict decode 与
migration，才能成为 `PIRDocument`。

## Canonical 来源

```text
PIR-current.version.json
  -> explicitly selected immutable wire snapshot
  -> PIR-current.json + backend current-version constant
  -> generated PIRWire* contract
  -> @prodivix/pir codec / migration
  -> unversioned PIRDocument
  -> semantic validator
  -> Workspace / Renderer / Compiler / Web
```

1. `PIR-current.version.json` 是唯一显式 activation manifest；不得扫描目录或按最高版本
   推断 current。
2. `packages/shared/scripts/sync-current-pir-schema.js` 只按 manifest 同步 current wire
   Schema 与后端常量。
3. `packages/shared/scripts/generate-types.js` 只从 `PIR-current.json` 生成唯一一份
   `packages/pir/src/codec/pirWire.generated.ts`；generated contract 不属于 shared
   package 的公开类型面，也不得生成或导出领域模型。
4. `@prodivix/pir/wire` 拥有 version dispatch、codec 与 migration；`@prodivix/pir` 默认入口
   拥有无版本 `PIRDocument`、normalizer、mutation、projection 与 semantic validator。
5. 非 codec/persistence 模块不得导入 generated wire type。
6. Canonical Workspace 只保存无版本 current model；transport / persistence adapter 负责
   current wire encoding，并且只向领域消费者暴露 decoded current model。

`PIR-v1.0.json` 至 `PIR-v1.2.json` 是 pre-canonical archival snapshots，不构成生产导入
承诺。`1.3` 是受支持 migration chain 的最低 canonical baseline；此后每个 activated wire
version 都必须保留到 current 的完整 migration path，缺链时 strict decoder fail-closed。

## 校验链路

读取链路：

```text
unknown persisted value
  -> read wire version
  -> strict version decoder
  -> deterministic migration chain
  -> current wire validation
  -> decode to version-neutral PIRDocument
  -> PIR semantic validation
  -> Workspace final-state validation
```

写入链路：

```text
Blueprint / AI / Plugin proposal
  -> PIR Command or atomic Workspace Transaction
  -> PIR semantic validation
  -> Workspace final-state validation
  -> local History
  -> WorkspaceOperation / Durable Outbox / Atomic Commit
  -> current wire encoder
  -> persisted document
```

校验至少覆盖：

1. closed wire shape、基础字段类型与 current wire version。
2. normalized graph identity、root、parent、region 与引用完整性。
3. Component、Collection、Route、NodeGraph、Animation、Code、Token 与 Asset 的类型化跨文档
   引用。
4. Renderer 与 Compiler 对同一 current projection contract 的语义一致性。
5. blocking diagnostic 的稳定 code、target 与 SourceTrace。

## Contract 升级

数字 wire version 使用固定的边界升级流程：

1. 新增下一份不可变 `PIR-v<version>.json` snapshot。
2. 在 codec 边界增加从已激活版本到新 snapshot 的确定性 migration 与轻量边界 fixture。
3. 运行 `pnpm run pir:activate-wire -- <version>`，原子更新 activation manifest，
   同步 `PIR-current.json` 并重新生成 `PIRWire*` 与后端 current schema/constants；任一
   生成、migration/codec 测试或边界检查失败时回滚这些派生产物。
4. 命令必须验证旧 wire fixture 可迁移到 current，并且 current encode/decode deterministic。

以上命令只激活 repository contract。生产发布还必须通过
`39.pir-current-evolution.md` 定义的 Canonical persistence rollout gate：先完成 backend
coordinated migration，或在首次写入时以同一 Atomic Commit 完整替换旧 wire document 后再
应用路径 patch。仅在前端 decode 旧文档而让后端继续保存旧 wire 时，路径级 commit 必须
fail-closed，不得假设保存态已经是 current。

若升级仅改变 wire 表达，Workspace、Renderer、Compiler、Semantic Index 与 Web 保持不变。
若升级新增领域语义，则直接扩展无版本 current model 和真正消费该语义的
稳定 API。

普通 wire 升级以生产消费者 diff 为空作为默认验收结果；改动通常只落在新 snapshot、
activation manifest、generated wire contracts、migration 与边界测试。

## 长期不变量

1. 生产领域入口只有无版本 `@prodivix/pir`。
2. 数字版本只存在于 wire schema、codec、migration、transport 与 persistence 边界。
3. PIR graph 保持 normalized；树只作为 Renderer 与作者表面的临时读投影。
4. PIR 写入经过 Command / Transaction、History、Durable Outbox 与 Atomic Commit。
5. Workspace Semantic Index 索引 PIR identity、scope 与 reference；PIR 保存态继续持有类型化
   领域引用。
6. 任意版本升级都保持单一无版本领域入口、单一 current 生产链与单一作者态真相。

## 验收标准

- [x] current wire Schema 与版本可由显式 manifest 确定性生成。
- [x] generated contract 使用 `PIRWire*` 命名并与领域类型隔离。
- [x] 无版本 `PIRDocument` 不保存数字 wire version。
- [x] codec 在 encode/decode 边界注入和剥离 wire version。
- [x] migration registry 已形成集中版本升级入口。
- [x] Workspace、Renderer、Compiler、Semantic Index 与 Web 统一消费无版本 PIR-current API。
- [ ] G1 Golden journey 使用同一 PIR-current 语义完成 Preview、Export 与保存恢复验证。

## 决策摘要

Prodivix Core 定义稳定 extension points、capability、Host lifecycle、Gateway、diagnostics 与 conformance；具体外部库、adapter、模板、runtime policy 与 codegen policy 由插件贡献。

Official 与 community plugins 使用同一个 Manifest、package、permission、sandbox、Host transaction 与 contribution query contract。所有插件作者态写入由 Host 调用领域 planner，形成 Command / Transaction 与 WorkspaceOperation。

## Package 与 Manifest

插件 package 至少包含：

```text
plugin package
  ├─ manifest
  ├─ versioned contribution resources
  ├─ verified runtime artifact（可选）
  ├─ static assets（可选）
  └─ integrity / publisher metadata
```

Manifest 声明：

1. stable plugin id、publisher、version 与 engine range。
2. activation events 与 runtime entrypoint。
3. requested capabilities、scope、reason 与 optionality。
4. versioned contribution resource references。
5. package integrity 与资源大小边界。

Manifest 只描述插件本身。项目级启用状态、插件配置与 contribution binding 位于 Canonical Workspace VFS 或 Workspace settings 的对应 owner 中。

## Extension point families

| Extension point          | 稳定职责                                                                |
| ------------------------ | ----------------------------------------------------------------------- |
| `paletteContribution`    | Blueprint palette item、分类、搜索 metadata                             |
| `blueprintTemplate`      | 可验证的 fragment/template materialization recipe                       |
| `externalLibrary`        | package coordinate、runtime types 与支持矩阵                            |
| `renderPolicy`           | runtime component、style、overlay 与 portal projection                  |
| `codegenPolicy`          | target import、element path、children policy 与 dependency contribution |
| `iconProvider`           | icon catalog 与 runtime/codegen mapping                                 |
| `inspectorContribution`  | 稳定 Inspector section/field contribution                               |
| `nodeGraphExtension`     | node/port/executor/diagnostic contribution                              |
| `animationExtension`     | track/effect/runtime/diagnostic contribution                            |
| `codeAuthoringExtension` | CodeSlot、Language Service、adapter 与 code action contribution         |
| `diagnosticProvider`     | revision-aware diagnostic provider snapshot                             |
| `workspaceDocumentType`  | document codec、validator、projection 与 capability                     |
| `importExportProvider`   | format reader/writer 与 ExportProgram contribution                      |
| `aiContextProvider`      | scoped context、tool proposal 与 provenance contribution                |

每个 extension point 独立版本化。Manifest 只声明 point/resource；Host exact contract registry 决定当前支持的 contract version。

## Host lifecycle 与 transaction

`@prodivix/plugin-host` 持有 transport-neutral Host Core：

1. package discovery 与 integrity validation。
2. immutable permission snapshot、deny-wins policy 与 live revoke。
3. owner/generation/lifetime-aware contribution registry。
4. staged atomic contribution transaction。
5. availability/runtime 双轴 lifecycle、operation serialization 与 cleanup。
6. runtime adapter port、structured audit 与 diagnostic bridge。

Contribution transaction 先完成 resource decode、Schema、semantic cross-reference、capability 与 implementation binding 校验，再以一个 owner generation 发布 immutable snapshot。

## Browser sandbox 与 Gateway

`@prodivix/plugin-browser` 提供 browser adapter：

```text
verified plugin artifact
  -> opaque sandbox origin
  -> Dedicated Worker runtime
  -> versioned MessagePort protocol
  -> Capability Host Gateway
  -> audited Host services
```

安全边界包括：

1. 独立 sandbox origin、CSP 与 Permissions Policy。
2. exact-version protocol envelope、sequence、correlation、timeout 与 cancellation。
3. heartbeat、message quota、request quota 与 deterministic termination。
4. live capability grant、owner/generation/session binding。
5. bounded persistent audit 与 network policy。
6. build-attested official Host implementation binding。

## Workspace 写入

插件通过 Host Gateway 请求产品动作：

```text
plugin request
  -> capability + scope + audit checks
  -> domain planner
  -> reversible Command / atomic Transaction
  -> Workspace + domain validation
  -> WorkspaceOperation
  -> Durable Outbox / Atomic Commit
```

规则：

1. 插件 runtime 不直接修改 Workspace snapshot、document、Outbox 或 transport。
2. PatchOp 只存在于 Host 生成的 Command 内部。
3. write capability 的 scope 与具体 target/operation 共同校验。
4. plugin id、package generation、capability revision 与 operation identity 进入 audit metadata。
5. conflict resolution 基于最新 canonical snapshot 生成新的 WorkspaceOperation。

## Code Authoring 与 Semantic Index

1. 插件的 executor、adapter、shader、mounted CSS 与其他 code-owned 能力通过 CodeArtifact、CodeReference 与 CodeSlot 接入 Code Authoring Environment。
2. 插件领域 provider 向 Workspace Semantic Index 发布完整 symbol/scope/reference/dependency facts。
3. Workspace Semantic Index 统一应用 identity、visibility、resolution 与 impact policy。
4. 插件 semantic diagnostics 作为 provider snapshot 进入 `@prodivix/diagnostics`。
5. AI context/tool contribution 消费稳定 query 与 Action Proposal contract。

## Official component plugins

Ant Design、MUI 与 Radix 是当前 bundled official component plugins。它们使用：

1. 真实 Manifest 与 versioned JSON contribution resources。
2. exact package/dependency coordinates 与 support matrix。
3. build-attested React Host Module。
4. Palette、Blueprint Template、External Library、Render Policy、Codegen Policy 与可选 Icon Provider。
5. workspace-scoped Host transaction 与 immutable query snapshot。
6. owner-scoped style/overlay lease、cleanup 与 audit。

四种 official export fixture 已覆盖单库和组合依赖的 install、build 与 browser behavior gate。Workspace 默认按项目显式启用外部库。

## 长期不变量

1. Core 认识 extension point，不认识每个外部库。
2. Official 与 community plugins 使用同一 Host contract。
3. Manifest 与 contribution resources 是纯数据、严格版本化并受 package integrity 约束。
4. runtime code 在 capability sandbox 中执行。
5. 插件写入统一进入 Domain Command / Transaction、WorkspaceOperation、Durable Outbox 与 Atomic Commit。
6. code-owned 能力统一进入 Code Authoring Environment。
7. 跨领域 symbol/reference/diagnostic 统一进入 Workspace Semantic Index 与 Diagnostics。
8. Workspace VFS 或 Settings owner 保存项目级插件状态。

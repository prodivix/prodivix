## 背景

finite query/mutation/request-reply 只产生一个结果，不能表达 GraphQL subscription 或 AsyncAPI receive/stream。
若 adapter 私自返回 event emitter，会绕过 Data schema、Execution authority、取消、预算、Network correlation
与 Session SourceTrace，也无法证明 iframe 消费速度对上游形成背压。

server/edge protocol gateway 还必须复用已经审计的 Remote Preview authority、Workspace permission、exact
environment revision、SSRF/HTTPS policy、response budget 与 sanitized trace，不能为 GraphQL/AsyncAPI 新建一条
弱化的出站路径。

## 决策

### 1. Canonical operation 与有限策略

- `DataOperationKind` 增加 `subscription`；它仍是 canonical DataSourceDocument 的 operation，不产生第二套 stream 文档。
- subscription 禁止 finite cache、retry、pagination、optimistic 和 invocation-key policy。断线后自动 replay 或把事件缓存
  当作 canonical state 均 fail closed。
- finite `executeDataOperation` 不接受 subscription；只有显式 `openDataOperationStream` 与声明 `openStream` 的 adapter 可执行。
- 当前支持 GraphQL subscription，以及 AsyncAPI `receive`/`stream` 的 HTTP SSE/NDJSON frozen subset。HTTP adapter 本身继续 finite-only。

### 2. Pull-driven stream contract

transport-neutral session 固定为：

```text
open(invocation) -> network metadata
pull(requestId, cursor N) -> event cursor N+1 | complete cursor N | safe error
cancel(requestId) -> close upstream and release identity
```

- iframe 每消费一个事件才发送下一次 pull；父窗口每次 pull 最多解码一个 NDJSON record。Backend 不构造
  application-level 全量事件数组，而以逐 record flush、TCP backpressure 和固定总预算约束传输；浏览器/内核的
  有界 transport buffer 不被描述为 durable cursor 或 exact one-record upstream pull。
- 同一 requestId 只允许一个 active stream 和一个 pending pull；cursor gap、duplicate open/pull、并发 read 或 generation drift
  都 fail closed。
- 单次会话最多 256 events、每个事件 256 KiB、总事件 4 MiB、最长 5 分钟、idle 30 秒；Data kernel 只保留最近
  64 个事件作为可丢弃 snapshot。
- close、AbortSignal、active Job replacement、iframe dispose 和 execution stop 都必须向上游传播取消并释放 identity。

### 3. Backend protocol gateway

- finite HTTP/GraphQL/AsyncAPI 共用 Remote Data gateway 的 execution-bound authority、Workspace read、environment revision、
  mutation permission/replay、HTTPS/SSRF 与 sanitized Network policy。
- GraphQL gateway 校验 exact named query/mutation/subscription document；AsyncAPI gateway只接受 frozen
  publish/request-reply/receive/stream action。
- Canonical Backend Workspace validator 接受 subscription current contract，并拒绝其 cache、retry、idempotency、
  pagination 与 optimistic finite policy；不能只在执行测试 fixture 中绕过 durable authoring Gate。
- upstream stream 只接受 `text/event-stream` 或 `application/x-ndjson`/`application/ndjson`，Backend 向 Web 输出严格
  `application/x-ndjson` envelope，不透传 header、endpoint path、GraphQL error message 或协议私有 payload。
- 本 ADR 的 first vertical 只允许 public stream；Secret/reconnect/incremental 边界已经由 ADR 55 修订为显式
  per-connection renewal、HMAC checkpoint 与 keyed collection contract，不能由 adapter 私有扩展。

### 4. Remote frame 与 provider capability

- generated runtime 只发送 value-only document/operation/invocation/input；access token、session、environment/Secret identity
  均不进入 capability iframe。
- Web 只接受 exact active frame、opaque origin、active Job/Session generation 的 open/pull/cancel；Backend bearer 只存在于
  product-authenticated fetch adapter。
- 含 subscription 的 Executable Snapshot preview requirements 必须包含 `data-stream`、`network` 与
  `environment-binding`。Remote Preview descriptor显式提供 `data-stream`；Browser/static client 和 mock-only stream 当前 fail closed。

### 5. SourceTrace debugger journey

- Backend 每个 finite/stream protocol Network trace 都附加唯一 metadata-only `data-operation` SourceTrace；wire decoder
  校验 trace/correlation 的 documentId、operationId、invocationId、sequence、attempt 与 adapter 一致。
- Remote coordinator 只把 opening Network metadata 发布到 exact terminal-capable Execution Session，不以 stream event
  复活 finite Job。
- Execution Center Network 同时提供 Inspector link 与 Source link。Source link 先校验 producing snapshot 等于当前
  Workspace snapshot，再校验 canonical Data document/operation 仍存在，最后通过共享 semantic navigation 打开 Resources。
- stale snapshot、缺失/歧义 SourceTrace、未知 source owner 或导航失败只返回稳定 unavailable reason，不猜测最近文件。

### 6. First vertical 明确不承诺

- 断线 resume cursor、Secret-authenticated connection 与 collection incremental merge 已由 ADR 55 在同一 execution
  的有界范围内补齐；跨 execution replay、durable event history、WebSocket、GraphQL WS、Kafka/MQTT 仍不承诺。
- 自动 durable UI binding 与 G3 Behavior/Verification Evidence 不由本 ADR 或 ADR 55 提前定义。

## 验收 Gate

`pnpm run verify:g2:data-stream-debugger` 必须覆盖：

1. Data pull/backpressure/cursor/byte/event/duration/abort contract；GraphQL 与 AsyncAPI adapter frame mapping。
2. Backend Canonical Workspace subscription persistence、finite GraphQL/AsyncAPI 与 SSE/NDJSON stream、authority、
   identity conflict、strict envelope 和 SourceTrace。
3. Runtime Core strict open/pull/cancel/event wire；generated server/edge runtime真实 postMessage journey。
4. Remote Web streaming fetch、active Job generation/cancel fence、Network Session publication。
5. React/Vue executable target capability Golden matrix，以及 client/HTTP/缺失 renewal Secret stream fail-close；ADR 55
   另覆盖显式 renewal Secret stream。
6. Execution Center Network Source link、exact snapshot fence 和 Data operation semantic navigation。

## 后果

- subscription 成为一等 Data runtime capability，但仍不会把无限事件流伪装为 finite lifecycle result。
- server/edge GraphQL/AsyncAPI 不再是 HTTP gateway 的特例分支，而是在同一安全边界下拥有协议 decoder。
- SourceTrace debugger 从 Server Function 扩展到 Data Network；新的 producer 应复用通用 source navigation input/result。
- ADR 55 的 same-execution reconnect/resume与credential renewal不等于 durable或跨副本 resume；二者必须在状态和证据中分开描述。

## 背景

G2 已有Browser/Remote Test、Vitest canonical report、React/Vue export、Browser/Remote Preview、build和Chrome product Gate。
G3需要把unit/integration/E2E/visual/accessibility/performance/security结果放进同一Plan/Evidence体系，并证明同一
BehaviorScenario在Preview、Export和CI运行。若Web直接理解Playwright/axe/Vitest私有JSON，或每个target维护自己的
Scenario脚本，G2建立的provider-neutral边界会失效。

## 决策

### Adapter registry

`@prodivix/verification`定义internal controlled adapter contract：

```ts
type VerificationAdapterDescriptor = Readonly<{
  id: string;
  version: string;
  checkKinds: readonly VerificationCheckKind[];
  surfaces: readonly ('preview' | 'export' | 'ci')[];
  targets: readonly string[];
  capabilities: readonly string[];
  trustInputs: readonly string[];
}>;
```

Adapter接收immutablePlan check、Scenario Program、Executable Snapshot/artifact refs、control profile和abort signal，输出
bounded `VerificationCheckReportCandidate`。Core负责strict normalization、identity、budget、SourceTrace和Evidence intake；
Web/Backend不解析工具私有payload。

G3 registry只开放first-party/controlled adapters，不承诺public SDK、第三方plugin或Marketplace；这些属于G6。

### Check families

#### Diagnostics/build/unit/integration

- diagnostics消费exact Semantic/Compiler/Workspace diagnostic snapshot；
- build复用G2 Build ExecutionJob和artifact digest；
- unit/integration消费canonical `ExecutionTestReport`，Vitest私有JSON仍由`@prodivix/runtime-vitest`转换；
- report file/case映射到stable check id与SourceTrace，大量原始详情作为artifact。

单个G2 report不是Evidence。Verification adapter必须补齐Plan/check/snapshot/tool/provenance并通过Evidence intake。

#### E2E/Behavior

Browser scenario driver消费同一`BehaviorScenarioProgram`。target定位优先使用accessible role/name、Route/public contract和
opaque semantic target id；禁止把CSS/XPath保存回Scenario。

Black-box observation（URL、accessible state、DOM-visible behavior、network metadata、screenshot）是跨surface closure的
主要证明。verification-only white-box probe只提供sanitizeddomain trace/SourceTrace诊断增强：

- probe由Compiler从同一ExportProgram生成；
- 只在verification/test profile激活；
- 没有Workspace/network/Secret/write capability；
- production artifact必须通过hard-cut检查确认未包含probe control endpoint；
- white-box trace不能成为唯一passed依据。

#### Visual

Visual adapter在controlled browser image、font、viewport、DPR、color/motion和checkpoint settle下capture PNG。Baseline
compatibility key至少包含Scenario/checkpoint、target、browser engine、OS/image、font set、viewport/DPR、color/motion和
renderer generation。

结果记录exact baseline/current/diff digest、changed pixel/region summary、threshold和mask/exemption。React/Vue或不同engine可
使用不同baseline，但共享semantic assertion；不能以跨framework像素完全相同作为行为parity。

Mask必须绑定stable semantic target/region并经Policy审阅，不能在failure后自动扩大。Baseline update产生Workspace Transaction。

#### Accessibility

Accessibility adapter将工具私有finding转换为canonical rule id、impact、target、message key、SourceTrace和bounded related node。
至少覆盖accessible name/role/state、focus order/trap、keyboard path、landmark/heading、contrast和dynamic announcement。raw DOM片段
不进入manifest；需要时以受控artifact保存。

Policy exemption保留raw finding与applied reason/expiry。跨engine规则结果按adapter schema compatibility比较。

#### Performance

Performance检查使用controlled environment采集navigation/interaction、resource、layout shift、long task、animation frame和
scenario duration aggregate。它验证声明budget，不把本机偶然毫秒差异当作跨环境回归。Evidence记录machine/image/browser/
warm-cache class和sample strategy；不兼容环境不做trend。

#### Security

Security adapter汇聚G2现有Secret canary、network allowlist、CSP/Permissions Policy、sandbox、dependency/static scanner和runtime
policy结果。它不在G3重新实现扫描器。Secret/canary命中、unexpected network、production probe泄漏和artifact digest drift是
不可豁免required failure。

### Surface与target matrix

稳定surface语义：

- `preview`：Editor Browser/Remote Preview，由本地或受控remote provider运行；
- `export`：从exact ExportProgram构建的standalone app/bundle，在独立browser/runtime运行；
- `ci`：非交互受控runner执行canonicalPlan，产生attested Evidence。

surface不是framework target。React/Vite与Vue/Vite必须消费相同Scenario/current domain；adapter capability差异在Plan中显式。

G3 Golden matrix采用风险分层，避免无界笛卡尔积：

| 范围                          | 必须覆盖                                                                            |
| ----------------------------- | ----------------------------------------------------------------------------------- |
| 所有critical Scenario         | Preview、Export、CI；primary engine；full/reduced（若有motion）                     |
| Authenticated Catalog CRUD    | React/Vite与Vue/Vite；loading/empty/error/retry/pagination/optimistic/mutation/auth |
| Cross-browser critical subset | Chromium、Firefox、WebKit的black-box semantic/a11y                                  |
| Visual checkpoints            | controlled compatibility cells，不要求跨target同baseline                            |
| Impacted code/domain          | unit/integration/build/diagnostics与policy-selected security/performance            |

每个cell独立报告unsupported/blocked；不能用一个target通过替代另一个required target。

### Tool identity与private payload

Adapter固定tool name/version、adapter schema version和normalized capability。私有JSON、browser handle、page object、SDK error、
absolute temp path和vendor URL不越过adapter。unknown field、oversized report、partial result和terminal-after-report drift fail closed。

Tool upgrade造成normalization或baseline compatibility变化时显式使历史comparison incompatible；不能悄悄复用旧baseline。

### Failure classification

规范化结果区分：

- assertion/finding failure；
- product diagnostic/build failure；
- unsupported capability；
- fixture/control failure；
- adapter/tool infrastructure error；
- cancellation/timeout；
- malformed/unsafe candidate。

只有第一类及受支持的policy finding属于产品失败；基础设施失败可有界重试但不能变成assertion passed。所有类别都能定位
Plan cell/check与适用的SourceTrace。

## 拒绝的方案

### Web直接解析Playwright/axe/Vitest结果

拒绝。工具schema会扩散，Remote/CI与未来adapter无法共享产品表面。

### 每个framework复制Scenario

拒绝。target adapter只负责projection；Scenario identity和semantic assertions共享。

### 为视觉一致强制React/Vue同一pixel baseline

拒绝。行为/可访问性应一致，framework/engine raster差异由compatibility cell管理。

### verification probe进入production

拒绝。production hard-cut必须证明控制endpoint、fixture和white-box bridge不存在。

## 后果

- 新建Verification Core和Browser adapter；受控Playwright/axe/image/performance adapter在private边界转换。
- Runtime/Compiler/Remote Worker需要verification profile和production probe hard-cut。
- G2 canonical reports/artifacts成为Evidence输入而非被替代。
- Golden conformance扩展为Scenario×surface×target×control matrix。

## 验收

- [ ] 所有tool private payload在adapter边界转换，Core/Web只见canonical candidate/report。
- [ ] black-box proof跨Preview/Export/CI，white-box probe仅作诊断且production hard-cut。
- [ ] visual baseline compatibility/mask/update、a11y finding/exemption和performance环境比较稳定。
- [ ] Secret/network/CSP/sandbox安全结果进入不可豁免Evidence Gate。
- [ ] React/Vue与三browser critical subset使用同一Scenario并有独立required cell。
- [ ] unsupported、partial、oversized、tool drift和infrastructure failure不会伪装passed。

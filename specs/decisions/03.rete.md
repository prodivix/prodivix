# [未来项] 使用 Rete.js 作为节点图编辑器

## 状态

- Superseded（Historical Candidate）
- 日期：2026-02-08
- 替代决策：`specs/decisions/30.react-flow-nodegraph-editor.md`

## 决策结论（历史方向）

Rete.js 曾作为 NodeGraph 编辑器的候选方向之一保留在 Parking Lot。

随着 NodeGraph 编辑器进入正式实现，长期技术选型已转向 React Flow（`@xyflow/react`）。本文档不再代表当前 NodeGraph 编辑器方向，仅作为历史候选记录保留。

## 被替代原因

1. 当前代码库已基于 React Flow 建立 NodeGraph 编辑器主链路，包括节点类型、连线验证、画布交互、PIR 序列化和执行器接入。
2. React Flow 已成为 NodeGraph 的长期选型，不再只是临时 PoC。
3. Rete.js 暂不进入主交付链路，也不作为默认实现目标。

## 本期边界

1. 不基于 Rete.js 实现 NodeGraph 编辑器 UI。
2. 不引入 Rete.js 运行时依赖到主交付链路。
3. NodeGraph 相关协议与实现以 `specs/decisions/30.react-flow-nodegraph-editor.md` 和 `specs/decisions/20.node-graph-port-semantics.md` 为准。

## 后续评估入口（仅保留）

仅当未来明确需要替换 NodeGraph 引擎时，才重新评估 Rete.js 或其他节点图引擎：

1. React Flow 无法满足长期性能、扩展点或许可要求。
2. 插件系统需要支持多个可替换 NodeGraph 引擎。
3. 新引擎能证明其迁移收益大于模型、测试和用户体验重构成本。

## 备注

本文档用于保留历史技术方向，不代表当前版本交付承诺。

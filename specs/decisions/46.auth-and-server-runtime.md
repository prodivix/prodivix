## 背景

G2 需要让导出应用拥有 Auth、session、permission、route loader/action/guard 与 server function，
同时保持 Canonical Workspace、Preview、Test、Remote 和 Export 的同一语义。产品自身登录态不能被
误当成导出应用的作者态，也不能把 cookie、Bearer token、session id 或 Secret 塞入 Workspace、
Executable Snapshot、iframe message、日志或客户端 bundle。

Route 已以 `loaderRef / actionRef / guardRef` 保存 `CodeReference`，Code Authoring Environment 已拥有
CodeArtifact 与对应 CodeSlot。因而 Server Function 不建立第二种源码文档或私有 function registry；
它必须继续由 canonical code document 保存源码，由 metadata 只声明可验证的运行 profile。

## 决策

### 1. Transport-neutral owner

`@prodivix/server-runtime` 是 Auth/Server 运行 contract 的唯一 transport-neutral owner，负责：

- `AuthPrincipal`、server-only `AuthSessionReference` 与 `AuthPermissionPort`；
- Server Function kind/effect/runtime zone/auth/schema/idempotency 与 reference-only environment policy current model；
- code metadata strict decoder、definition resolution、instance-owned adapter registry；
- authorization-before-effect、input/output JSON Schema 与 outcome kernel；
- bounded value-only Remote Preview bridge codec。

它不依赖 React、Gin、Browser、Remote transport、Workspace persistence 或特定身份供应商 SDK。

### 2. Canonical Server Function profile

Server Function 源码继续由 `code` Workspace document 持久化。该文档可以声明唯一 metadata key：

```ts
metadata['prodivix.serverRuntime'] = {
  schemaVersion: '1.0',
  functionsByExport: {
    loadPrincipal: {
      kind: 'route-loader',
      runtimeZone: 'server',
      adapterId: 'core.auth.current-principal',
      effect: 'read',
      auth: { kind: 'authenticated' },
      inputSchema: { type: 'object' },
      outputSchema: { type: 'object' },
    },
  },
};
```

profile 只接受 TypeScript/JavaScript、exact fields、1-128 个 named exports、JSON Schema 2020-12
与 `server | edge` zone。单个 schema 上限为 256 KiB、64 层、8192 nodes，只允许 document-local ref；
Backend code document 读取上限为 4 MiB。Route 仍保存 `CodeReference { artifactId, exportName }`；profile 不复制 artifact
identity，registry 不拥有作者态 binding。Workspace TypeScript validator、Compiler 与 Backend exact-revision
gateway 都 fail closed 解码该 profile。

需要 Secret 的审计 adapter 只在 function entry 保存 field -> `SecretRef { bindingId }`：

```ts
environment: {
  secretsByField: {
    key: { bindingId: 'webhook-signing-key' },
  },
}
```

该 map 必须有 1-32 个 canonical field，reference 只允许 exact `bindingId`；value、grant、session、token 与
environment snapshot identity 均不进入 profile。Transport-neutral kernel 只向 adapter 暴露声明 field 的
`useSecret(field, callback)`，在 bounded outcome validation 后执行 material canary Gate，并在成功、拒绝、取消或异常时
revoke resolution lease。未声明 field、缺 lease、额外 lease 与 material echo 均 fail closed。

### 3. Auth identity、session 与 permission

- `AuthPrincipal { providerId, principalId }` 是 adapter 可见的稳定身份投影，不包含 token、cookie 或
  session id。
- `AuthSessionReference { providerId, sessionId, principalId, expiresAt }` 只存在于可信执行宿主；
  authorization kernel 校验 provider/principal/expiry 后，绝不把 session 传给 adapter 或 iframe。
- auth policy 是 `public | authenticated | permission(permissionId)`。permission 必须通过注入式
  `AuthPermissionPort` 在 effect 前决定；静态 provider capability 不能代替逐次授权。
- permission request 绑定 workspace、principal、session、function reference 与 permission id，但不携带
  credential material。

生产 product session provider id 当前为 `prodivix-product-session`。这是 first vertical adapter identity，
不把产品登录系统定义成所有导出目标的永久 Auth provider。

### 3.1 Canonical reference-only Auth configuration

Workspace 以唯一 `/config/auth.json`、`project-config` document 保存可审计的 provider 与 permission catalog：

```json
{
  "schemaVersion": "1.0",
  "providerId": "prodivix-product-session",
  "permissionIds": ["workspace.owner"]
}
```

该 contract 只允许 exact fields、canonical id、sorted unique permission，最多 32 项；unknown field、token、
cookie、session、Secret 或未规范化 identity 均 strict decode 失败。它是作者声明，不是 principal、session、grant
或 permission decision。`AuthPermissionPort` 仍必须在每次 effect 前根据可信 authority 做判定。

受保护的 Route binding 在 Workspace authoring projection、React/Vite Compiler 和 isolated production Compiler
三层检查该 document。缺失、无效、所需 permission 未声明均 fail closed；Remote/isolated 当前只接受
`prodivix-product-session`。Deterministic Test 复用 canonical permission catalog，但仍使用 execution-only fixture
principal，不把测试 provider 写回作者态。Resources 的 Auth & Server Runtime 页面只通过可逆 Workspace Operation
创建或更新该 document，不提供任何 credential 输入框。

### 4. Invocation 与 outcome

每次调用绑定 exact function reference、invocation id、attempt 与 bounded JSON input。bridge input/outcome
各自上限为 1 MiB、64 层、65536 nodes，且 number 必须是有限值。稳定 outcome 是：

- `value`：loader/action/function 的 JSON value，并在 effect 后按 output schema 校验；
- `allow | deny(code)`：guard 的授权结果；
- `redirect(location, status)`：loader/action/guard 的同源绝对路径跳转。

guard 不得返回任意 value，普通 function 不得发起 redirect。mutation 可以声明
`idempotency: { kind: 'invocation-key' }`。deterministic Test target 已实现 invocation-key mutation
replay/conflict fence；Remote live gateway 现在只为审计过的 execution-state adapter 开放 mutation，使用
PostgreSQL 单事务提交 effect 与 replay result，不把一般项目源码 mutation 或跨系统 effect 误称为 exactly-once。

### 5. Remote authenticated first vertical

当前可执行链路为：

```text
generated Remote Preview iframe
  -> exact source Window + opaque capability origin + strict value-only message
  -> apps/web generation-fenced coordinator
  -> product-authenticated Backend request
  -> exact execution owner/session/snapshot/code content revision
  -> strict profile + input schema + built-in adapter + output schema
  -> strict value-only response
```

Backend 只支持两个内置 read adapter：

1. `core.auth.current-principal`：`route-loader`，返回 provider/principal identity。
2. `core.auth.require-workspace-owner`：`route-guard`，要求 `workspace.owner` permission 并返回 `allow`。

另有一个严格受限的 live mutation adapter：

3. `core.server.execution-state.put`：只接受 authenticated `route-action + server + mutation + invocation-key`，
   将 generated typed Route action 中的 `{ key, value }` 写入 exact execution/function partition，并返回
   `{ key, value, revision }`。它不访问网络、Secret、Workspace 写入链或项目源码；state/replay 随 execution
   authority 级联删除，单 execution 的 state key 与 replay 各自最多 256 条。

另有一个严格受限的 Remote live Secret read adapter：

4. `core.server.hmac-sha256`：只接受 authenticated `route-action + server + read`，并且 environment policy 必须
   恰好声明 `key -> SecretRef`。adapter 对 typed Route action 的 JSON submission value 做 deterministic canonicalization，
   只在 Backend Environment store 的 `UseSecret` callback 内计算 HMAC-SHA256，返回
   `{ algorithm: 'HMAC-SHA256', digest }`。Secret 长度、route envelope、output schema 与 material echo 都有硬 Gate；
   项目 source 不执行，也不接触 Secret。

Backend API 进程不 `eval`、import 或执行 canonical project source；source 仅用于 Code Authoring/未来隔离
server target，当前 adapter 行为由受审计产品代码实现。Remote execution grant 即使没有 environment
reference 也始终绑定创建它的 authenticated session，且 code document 必须匹配 snapshot partition revision。
HMAC vertical 还要求 execution authority 带 exact live environment id/revision；Backend 为
`prodivix.remote.server-function-gateway + sandboxed + trusted-service + server + process` 签发最多 30 秒且不超过
product session expiry 的 durable grant，绑定 workspace/principal/session/execution/artifact/export/invocation/binding/field，
callback 后立即 revoke。grant/use/revoke 由既有 Environment audit 持久化，material 不进入 bridge、snapshot、trace、日志或响应。

Remote Preview coordinator 在 exact active generation 完成调用后，将 `server.function` completion observation
发布到 exact `sessionId + jobId`；isolated production Worker 则只在 canonical result artifact 经二次校验并成功上传后，
向 durable Job event stream 发布同一 trace contract。`prodivix.server-function-invocation-trace.v1` detail 只保留
request/invocation/attempt/function identity、开始/完成/耗时、outcome、result kind 或安全 error code/retryable 与固定
`redacted: true`；input/output、principal、session、cookie、token、Secret 与源码均不进入 detail。Remote provider
要求 production success 同时存在唯一 result artifact 与随后发布的 trace，并强校验 request/span/function、artifact
status/error、最多 128 条 SourceTrace、唯一 root CodeArtifact 与 artifact/trace SourceTrace 一致性。

Session 可以在 finite Preview Job terminal 后继续有界保留 observation，但不得重新打开或改变 terminal Job。
Execution Center 的 Server 表面只消费 strict codec；仅当 SourceTrace 对被调用 CodeArtifact 有唯一归属时显示源码入口，
并在打开共享 Code Authoring overlay 前重算 exact Workspace snapshot。旧 snapshot/generation、错误 Session/Job、
冲突、helper-only/ambiguous source 或夹带未知字段均 fail closed。当前 producer first vertical 为 Remote Preview 与
isolated production，Browser 不伪造 Server observation。

### 6. Compiler 与 target matrix

Compiler 在 route runtime binding 处识别 Server Function profile并进行 client/server partition：

| 表面                         | 当前行为                                                                                                                                                                    |
| ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Browser Preview              | 缺 Server gateway 时 `WKS-EXPORT-SERVER-GATEWAY-REQUIRED`，fail closed                                                                                                      |
| Remote Preview               | 显式 execution parent gateway；read Auth、受限 execution-state mutation与审计 HMAC；HMAC 同时要求 `server-function`、`environment-binding` capability                       |
| Workspace Test               | 强制 deterministic-test target；fixture/principal/permission 缺失时 fail closed；Secret-bearing definition 不允许 fixture 降级                                              |
| ZIP/static Export            | 不生成依赖编辑器父窗口的伪 standalone 应用，fail closed                                                                                                                     |
| Remote Build                 | 可以构建 Remote Preview client artifact；build 本身不调用 Server Function                                                                                                   |
| full-stack production target | public\|authenticated\|workspace.owner permission/read bounded canonical TS/JS graph 与 sealed Secret code-export first vertical 已由独立 Remote Worker/rootless target执行 |

带 Server profile 的整个 Code document 都视为 server-owned，不进入 generated client module/file graph；
同文件中的 sibling export 也不得复制到客户端。App 只生成 function reference 与 bridge，profile 未声明的
named export 必须 compile blocked。
Route guard 在页面渲染前执行，deny/redirect/error 均阻止页面渲染；loader 在 guard 后执行。Generated
React/Vite runtime 已公开 typed Route action dispatcher，投影 route/params/search/submission input，支持
AbortSignal、navigation cancellation、redirect/error、invocation-key replay 与 value outcome 后 loader
revalidation。loader result 的完整 PIR/Data authoring consumption 继续在后续纵切完成。

Deterministic Auth fixture 是 execution-only、content-addressed snapshot provision，不写 Workspace。Snapshot
v6 的 Test filesystem 投影 `deterministic-test` envelope；同一 snapshot 用于 Preview/Build 时只投影
`disabled` envelope且不含 principal/fixture value，避免测试身份进入可交付 client artifact。

### 6.5 Isolated production first vertical

Snapshot v6 以 digest-bound `production` entrypoint 和 `serverFunctionPlan` 描述一次性执行，不在 request
或 provider 中复制源码。Compiler 当前生成 `public|authenticated|permission(workspace.owner) + read + server + prodivix.code-export` 的 bounded canonical
Node ESM graph：最多 128 个 TS/JS Code document、64 层和 4 MiB UTF-8 source，只解析相对 static ESM
import/export；exact path、extensionless 与 `.js -> .ts` fallback 必须唯一解析。模块按 canonical path/id 排序并改写为
确定性 `.mjs` topology，cycle 保留，每个 generated module 绑定自己的 CodeArtifact SourceTrace。package/Node builtin、
dynamic import、CommonJS require、import type expression、triple-slash reference、越界/缺失/歧义路径均在 snapshot 创建前
fail closed。其他 permission、mutation、edge 与未知 adapter 仍在 compile 阶段 fail closed。
`prodivix.remote.server-function` 是独立 provider identity，不共享 Preview/Test/Build Job。

Control Plane 只校验/保存 neutral snapshot并路由到专用 Worker，不加载源码。authenticated create 由 Backend
在 exact Workspace owner preflight 后，通过受信 transport metadata 附加最多 5 分钟的
principal/`workspace.owner` permission/Workspace/snapshot attestation；它不进入 public envelope、
request、snapshot 或 generated source。Control Plane 在 execution create 同事务写入独立 authority row，幂等 identity
绑定 principal/permission/target，claim 时再绑定 execution/worker/attempt，过期不投影且终态删除。permission list
严格排序、去重且最多 32 项；当前 Backend 只签发 `workspace.owner`。该 contract 不含 session id、
Bearer、cookie、service token、lease token 或 Secret。

Worker 只在 install 已结束、残留进程已清理且 runtime network-none 经 host 验证后，才删除并 mode-0700 重建 reserved
`.prodivix` transport directory、强制四个 canonical transport path，并通过 nonce-bound 第二阶段消息将 exact value-only request
写入 execution-local 文件，在 rootless、resource/output/timeout/cancellation/lease fence 下运行，
由可信 Server Runtime再次强制 public|authenticated|workspace.owner-permission/read/server/code-export policy。
protected path 只将最小 `AuthPrincipal` 与 allowed permission ids 写入 mode-0600 authority file；generated runner
校验 exact Workspace/snapshot/expiry/required permission 并在调用项目代码前
删除，public path 丢弃该 projection。再以原始 snapshot profile和 output schema
二次校验 untrusted result。只有 correlation、function identity、schema与 root/import graph source trace均匹配时才发布 canonical
result artifact；artifact 成功上传后再发布 metadata-only `server.function` durable trace，Remote provider 必须校验两者
同源后才接受 success。isolated code-export 现只对 read/server/public|authenticated|workspace.owner profile 解析
reference-only SecretRef：Worker claim 后生成临时 X25519 key，Control Plane 在 exact lease/attempt/snapshot fence 下请求
Backend internal broker，Backend 重新验证 execution/session/code/environment revision 并把 callback material直接密封给 Worker。
Control Plane 不接触明文；install payload 不含 invocation/authority/Secret，静态 marker 不能提前释放 material；Worker 只在
nonce-bound runtime phase 清除 installer 预创建文件/符号链接后写 mode-0600 one-shot material，runner 在 import/effect 前校验并删除，再以 `useSecret` callback
暴露声明字段，runtime 后再次清除残留进程才允许 filesystem capture。product credential、其他 permission、edge 与项目
源码 live mutation仍不开放。

旧 Worker lease 过期后，read-only isolated invocation 可以由更高 worker attempt reclaim。Control Plane 只在 exact active
`starting|running` lease 下重新请求；Backend resolution ledger 每个 execution 始终只有 current attempt 一行，并仅在
artifact/export/invocation identity 不变且 attempt 单调增加时原子清空旧 ciphertext、替换 recipient。superseded attempt 的
delayed complete/replay/abandon、同 attempt key drift 与跨 invocation drift都不能修改 current row。新 attempt 必须重新经过
exact environment revision grant并生成绑定新 recipient/attempt 的 envelope；`cancelling` reclaim 在 resolution 前终止。

### 7. CSRF、CORS、cookie、token 与 Secret

- iframe bridge 不接受或返回 authorization、cookie、session id、token、SecretRef material 或源码。
- product Bearer token 只存在于 Web composition 到 Backend API 的既有 authenticated HTTP port。
- Backend endpoint 复用产品 Auth middleware、same API CORS policy 与 no-store response；不把 cookie/token
  转发给 function adapter。live mutation 额外要求 exact configured Origin 与
  `X-Prodivix-Server-Function-Intent: mutation-v1`；缺失、跨 origin、跨 allowed-origin replay 或 credential echo
  均在 effect 前失败。
- `core.server.execution-state.put` 是当前唯一 Remote live mutation adapter。它以 execution/function/state-key
  隔离 durable state，以 `(execution, function, invocation)` + snapshot/code revision + origin + canonical input
  形成 SHA-256 replay identity，并在同一 PostgreSQL transaction 中提交 state revision 与 replay result。
  exact duplicate 返回原结果，identity drift、取消与容量耗尽 fail closed；deterministic Test action 不放宽该边界。
- Server Function input/output、diagnostic、Session trace、artifact 与 generated source 均不得包含 Auth
  session material或 resolved Secret。
- Remote live Secret 只允许 audited adapter + exact live Environment authority。Browser/static、deterministic Test 与
  其他 adapter 即使声明 SecretRef 也必须 compile/runtime blocked。isolated target 只允许固定 `prodivix.code-export`
  read/server policy，并必须经 exact worker-token + lease endpoint、ephemeral X25519 recipient、Backend
  `remote-isolated/isolated-runner` grant 与 ciphertext-only one-shot replay row；resolution 只在 `starting` phase 开放，
  Worker 使用 15 秒 timeout，Backend/Control Plane response 强制 JSON + `no-store` + `nosniff` 并在读取时执行
  768 KiB streaming hard cut，所有失败只投影固定安全原因；不得把 Secret、broker service token 或 environment grant
  塞入 Worker authority/claim/snapshot/request。
- Execution Center Server 表面不得从 bridge input/output 重新构造详情；只显示 strict sanitized trace 的
  function identity、attempt、result kind/安全 error code 与 duration。

## 拒绝的方案

### 在 Route/PIR 中保存裸 server function 源码

拒绝。它绕过 Code Authoring、CodeReference、Semantic Index 与 Workspace transaction。

### 把产品 access token 或 session id 发给 Preview iframe

拒绝。iframe 是不可信 client runtime，任何 token bridge 都会扩大 XSS、日志与 artifact 泄漏面。

### 在 Backend API 进程直接执行用户 TypeScript

拒绝。API 进程持有产品 credential 和数据库权限，不是不可信代码 sandbox。

### 静态导出时静默跳过 guard/loader

拒绝。它会把受保护路由降级为公开页面；缺少 server target 必须在 compile Gate 阻断。

## 决策摘要

进程内作者态、远端持久化、离线恢复与领域编辑统一围绕 `@prodivix/workspace` 的 `WorkspaceSnapshot` 工作。Web Store 持有一个 canonical snapshot；API、IndexedDB、Compiler、Renderer、Git 与 Project publication 通过明确 codec 或 projection 连接该 snapshot。

## Canonical snapshot

```ts
type WorkspaceSnapshot = {
  id: string;
  name?: string;
  workspaceRev: number;
  routeRev: number;
  opSeq: number;
  treeRootId: string;
  treeById: Record<string, WorkspaceVfsNode>;
  docsById: Record<string, WorkspaceDocument>;
  routeManifest: WorkspaceRouteManifest;
  activeDocumentId?: string;
  activeRouteNodeId?: string;
};
```

PIR、Route、NodeGraph、Animation、Code、Asset 与 Config 的作者态内容全部位于该 aggregate。Tree、Canvas、Timeline、Preview、Export、Git、Issues 与 Semantic Index 是可重建读投影。

## Wire 与 Store

1. `decodeWorkspaceSnapshot` 是 wire DTO 到 canonical snapshot 的入口。
2. `encodeWorkspaceSnapshot` 是 canonical snapshot 到 wire DTO 的出口。
3. Mutation response 先解码为 `DecodedWorkspaceMutation`，再按 revision 与 operation identity 应用。
4. Zustand Workspace slice 的权威作者态字段是 `workspace: WorkspaceSnapshot | null`。
5. capabilities、readonly、selection、viewport、panel 与 tab 是独立产品/UI 状态。
6. Settings 使用独立的 settings state、Durable Outbox 与 Atomic Commit contract。

## Revision 与本地编辑

1. `workspaceRev`、`routeRev`、`opSeq`、document `contentRev/metaRev` 表示 server-confirmed revision。
2. 本地未确认编辑使用 `documentEditSeqById`、History 与 pending operation identity 协调。
3. ACK 推进 confirmed revision、local replica watermark 与 operation state。
4. confirmed base 加 pending Operation/Settings Outbox materialization 构成离线作者态读取结果。

## 唯一写入链路

Blueprint、Inspector、drag/drop、Route、NodeGraph、Animation、Code 与 Resources 通过领域 planner 生成 `WorkspaceCommandEnvelope` 或 `WorkspaceTransactionEnvelope`。

Command / Transaction 经过 patch path policy、Workspace/Route/PIR/领域 validator 和 final-state validation，再进入 History、WorkspaceOperation、Durable Outbox 与 Atomic Commit。

## Route 与 active document

1. Route selection 通过 composed RouteManifest 解析 `pageDocId` 与 `layoutDocId`。
2. `activeDocumentId` 指向当前 canonical Workspace document。
3. route create/move/bind/unbind/delete 与相关文档/VFS 修改使用 Command 或 Transaction。
4. Editor surface 通过 selector 消费 route、document 与 active context。

## 加载与恢复 Gate

编辑器进入 authoring-ready 之前依次完成：

1. remote response 或 local replica record 通过 codec。
2. Workspace、RouteManifest、VFS 与领域 document validator 通过。
3. confirmed base 与 pending operations 确定性 materialize。
4. canonical snapshot 一次性进入 Store。
5. capabilities、readonly 与当前项目 UI context 完成装载。

加载诊断保留稳定 workspaceId、revision、operation identity 与 target，供 error surface、Issues 和恢复动作使用。

## Project 与 publication

1. Project 服务保存项目 metadata 与权限边界。
2. Backend Workspace 保存 canonical authoring aggregate。
3. 社区、模板与发布能力通过显式 publish 从指定 Workspace revision 生成 publication projection。
4. Project metadata 与初始 Workspace 在一个数据库事务中创建。

## 长期不变量

1. 一个进程内作者态只存在一个 canonical WorkspaceSnapshot。
2. Wire DTO 只存在于 API/IndexedDB adapter 边界。
3. confirmed revision 与本地 edit sequence 使用不同语义。
4. localStorage 只保存主题、选择、viewport、panel 与 tab 等 UI 偏好。
5. 领域状态浏览器持久化由正式 local replica 与 Outbox adapter 承担。
6. Project publication projection 只服务显式发布场景。

## 验收标准

- [x] Web、API、IndexedDB 与 Core 使用统一 Workspace codec。
- [x] Zustand 持有一个 canonical Workspace snapshot。
- [x] confirmed revision 与本地 edit sequence 已分离。
- [x] Command / Transaction、History、WorkspaceOperation、Durable Outbox 与 Atomic Commit 已接入生产写入。
- [x] route selection 与 active document 使用 canonical RouteManifest/document identity。

## 背景

G2 要求 React/Vite 与一个受控第二 framework target 通过同一 CRUD runtime journey，以证明
`ExportProgram`、Data current model、Executable Project Snapshot、Browser/Remote Runner 和 Test contract
没有被 React 私有实现锁死。这个要求不是 G6 的多框架 Target SDK，也不授权在缺少语义映射时把
React source 转译成另一种框架源码。

实现前，Compiler 的通用 planner 已声明 `vue` target，但只有 React/Vite scaffold、Workspace compiler
和 executable snapshot generator。Data standalone runtime 本身是 framework-neutral TypeScript，只是历史上
位于 `react/` 目录。第二 target 应直接复用这个 runtime 与 canonical Workspace document，而不是复制一套
Vue 私有 Data lifecycle、cache、retry、mock 或 mutation executor。

## 决策

### 1. G2 选择 Vue 3 + Vite

第二 target 固定为：

```ts
{
  presetId: 'vue-vite';
  framework: 'vue';
  runtime: 'vite';
}
```

选择原因是 Vue SFC 能提供与 React/TSX 不同的 framework/compiler surface，同时继续使用独立 Vite
install/typecheck/test/build/browser-smoke Gate。本文冻结最初的 controlled Data subset；PIR/Route/Auth/Server/Asset
产品扩展由 ADR 54 单独约束，NodeGraph/Animation component integration 和公共 Target SDK 不因此自动成为已支持能力。

### 2. Canonical 输入与共享 runtime

- 输入只能来自 exact revision 的 `WorkspaceSnapshot`；Data Source/Schema/Operation/Policy 继续使用
  `@prodivix/data` current model。
- Vue target 直接生成并消费与 React target 字节一致的 `prodivix-data-runtime.ts`。它不得 fork lifecycle、
  schema validation、retry、cache、optimistic mutation、mock fixture 或 live HTTP/GraphQL/AsyncAPI finite policy。
- target-specific 代码只拥有 Vue scaffold、operation manifest、显式 Test Operation surface 与 component glue。
- query 排在 mutation 前；页面允许 query 自动 activation，但 mutation 只能由显式用户动作触发。
- Workspace response、cache、lifecycle 和 fixture session 仍是可丢弃运行态，不写 Workspace/Outbox。

### 3. Controlled document boundary

first vertical 接受 `data-source`、reference-only `asset` 与 `project-config` document。其他 document type 产生
稳定 `VUE-TARGET-DOCUMENT-UNSUPPORTED` error 并阻止 executable snapshot；不得静默丢弃、嵌入 React
产物或以 JSON renderer 冒充语义等价。

Binary Asset 继续通过共享 materialization/resource planner 做 exact-byte 验证。Vue PIR/Route/Auth/Server/Asset
产品旅程必须由 ADR 54 的独立 Golden 证明，不能由本 ADR 的 Data portability Gate 代替。

### 4. Data zone 与 Secret hard cut

- 默认 target 是 `static-client`，只允许可安全导出的 HTTP、finite GraphQL 与 finite AsyncAPI client Data。
- 缺少 execution-parent gateway 的 server/edge HTTP/GraphQL/AsyncAPI、client SecretRef 与受保护 client
  environment binding 在 standalone Vue compile 中 fail closed。显式 execution-parent target 可投影与 React
  字节一致的 finite gateway，以及 ADR 49/55 定义的 public或显式 per-connection renewal GraphQL/AsyncAPI bounded
  stream；stream必须要求`data-stream` capability，缺失renewal的Secret stream继续fail closed。
- generated source、package metadata、diagnostic、snapshot files 和 browser bundle 不得包含 Secret material。
- mock provision 只保存在 Executable Snapshot 的 typed runtime provision；reserved runtime files由 provider
  在 Preview/Test materialization 时投影，不能成为第二套 Workspace truth。

### 5. Executable 与 Remote parity

Vue target 使用同一 `prodivix.executable-project.v6`：

- Preview、Build、Test 使用现有 provider-neutral entrypoint/capability plan；
- mock fixture、content digest、source trace 和 dependency fingerprint 进入同一 snapshot normalization；
- Remote strict codec 不添加 framework 分支，只往返保存 target facts 和 exact files；
- Browser/Remote provider 不解析 Vue 私有 result，仍只消费 canonical Job event、artifact、trace 和 Test report。

### 6. 退出 Gate

controlled target 必须同时通过：

1. Compiler conformance：React/Vue 输出中的 standalone Data runtime 完全一致；unsupported/Secret fail closed。
2. HTTP/GraphQL/AsyncAPI mock/live protocol × React/Vue target snapshot matrix，以及 Preview/Test/Build runtime
   policy、Remote codec、provider capability round-trip、受审计 server/edge gateway 与未授权 client/Secret stream denial。
3. 独立工程：fresh install、`vue-tsc --noEmit`、Vitest、production build。
4. 真实浏览器：同一 mock collection 完成 list/get/create/update/delete、automatic query revalidation、
   loading/empty/retryable error attempt 与 pagination。
5. GitHub Gate：`.github/workflows/g2-data-closure.yml` 在相关路径变更时重复运行。

## 后果

- G2 获得第二 framework 的真实可执行 portability proof，而不是仅有类型枚举或 scaffold 文件。
- shared Data runtime 的语言基线固定为 ES2022；两个 target 后续必须同步该基线。
- 受控 subset 之外的 Workspace 会明确阻断；ADR 54 只把当前 canonical 产品 contract 接入 Vue，不扩张为通用 Vue SDK。
- G6 若建立公共 Target SDK，应复用本 ADR 的 target identity、planner 和 Gate，但需另行接受能力/版本/生态契约。

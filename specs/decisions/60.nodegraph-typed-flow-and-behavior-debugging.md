## 背景

G2 NodeGraph 已有 current document、deterministic executor、registry、step budget、same-context provider、Session、
SourceTrace 与 editor/Blueprint trigger。但默认节点仍主要验证 start/process/switch/log/end，缺少跨领域 typed data、
async/error/cancel、subgraph、CodeSlot 和可重复单步；因此 standalone export 不能替代真实 invoked behavior parity。

G3 要扩展 NodeGraph，但不能让 graph 成为任意网络、Secret、Workspace patch 或分布式任务调度器。

## 决策

### Current model hard cut

NodeGraph 继续由 `@prodivix/nodegraph` 独立拥有。扩展 current model时：

- port 必须有 stable id、direction、control/data kind、typeRef、required/multiple；
- edge 必须绑定 exact source/target port，旧仅 node-level edge 经 migration 或 fail closed；
- node descriptor 声明 executor id/version、input/output contract、effect class、runtime zone 和 capability；
- editor metadata 与 runtime semantics 分离；
- Code executor只保存 `CodeSlotBinding`，不保存裸源码/closure。

数字 wire version不进入生产 API。Backend、Workspace、Semantic Index、Compiler 与 package codec必须 conformance-equivalent。

### Typed control/data flow

control edge决定执行顺序；data edge传递 immutable transport-safe value。执行前 planner 完成：

1. port/reference resolution；
2. type assignability、required/multiple 与 cycle policy；
3. runtime zone/capability/effect planning；
4. Data/Route/Animation/CodeSlot target revision解析；
5. deterministic execution plan和digest。

隐式读取“最近一个节点输出”、按 label匹配端口或在 executor内扫描 graph 均禁止。数据 cycle只通过显式
loop/state/delay node表达；control cycle只通过有预算的 loop/subgraph contract表达。

### Node capability classes

G3 first-party registry至少覆盖：

- pure transform/branch/merge；
- temporary application state read/update；
- Data query/mutation invocation；
- Route navigate/param observation；
- Animation play/control/marker wait；
- registered CodeSlot invoke；
- structured log/assert/checkpoint；
- explicit async wait, retry, timeout, error boundary and cancel scope；
- subgraph/function call。

每个副作用节点复用领域 owner API。Data mutation 使用 Data dispatch/idempotency，navigate 使用 Router，Animation
使用 effect lease，CodeSlot 使用 Code Authoring/Execution capability。Node executor不能直接 fetch、读 environment、解析
Secret或修改 Workspace。

### State 与 transaction

NodeGraph state只表示本次 application/runtime context中的 temporary state。`RuntimeStatePatch` 不等于 Workspace Patch。
一组 runtime state action可以作为 graph transaction原子提交到临时 state host；失败时回滚该 runtime transaction。

任何采纳作者态的结果都必须由领域 planner显示 impact/diff并创建 Workspace Command/Transaction。Verification run永不
自动采纳。

### Async、error、cancel 与 retry

- async node返回明确 outcome，不泄漏 Promise/AbortController；
- cancellation signal沿 scope/subgraph传播，terminal后迟到 completion被generation/attempt fence丢弃；
- error edge接收 sanitized typed error，不接收任意 exception object/stack；
- retry仅用于声明可重试且effect/idempotency contract允许的节点；
- timeout、retry schedule与concurrency lane使用 Scenario deterministic scheduler；
- parallel branch有显式join/merge policy，输出排序不依赖完成时刻。

### Subgraph 与 graph function

Subgraph通过 document-qualified reference和显式 input/output public contract调用。编译时检测missing、cycle、recursion depth、
capability escalation和revision drift。默认禁止递归；未来若开放，必须有显式递归节点和深度/step budget。

调用不复制子图，也不按 Web store读取。Compiler、Preview、Remote/Export使用同一 resolved graph program。

### CodeSlot executor

CodeSlot声明language/capability、input/output type、purity/effect、runtime zone和budget。resolution绑定 CodeArtifact revision、
symbol/source span和implementation digest。unknown type、stale artifact、capability mismatch或non-serializable output fail closed。

test-zone custom node默认无network/environment/Secret/Workspace write。需要Data/Server effect时必须调用共享typed gateway，
不能由CodeSlot自行fetch。

### Debug protocol

NodeGraph debugger复用deterministic execution plan和scheduler，提供：

- breakpoint（node enter/exit、edge、error、checkpoint）；
- pause/step-into/step-over/step-out/continue/cancel；
- bounded call stack、active scope、port value和temporary state projection；
- trace timeline、Scenario step/barrier correlation与SourceTrace navigation；
- replay divergence定位。

debug command绑定job/attempt/plan/generation和lease；stale command fail closed。值按type、depth、count、UTF-8 byte和Secret
redaction投影。breakpoint/selected frame/expanded value是runtime UI state；保存断点或assertion要走Command。

### Execution topology

完整项目在Remote/Export运行时，NodeGraph program随Executable Snapshot在同一sandbox/client runtime执行。G3不按节点
做远程RPC或分布式调度。privileged node通过G2 Data/Server gateway；provider必须满足program aggregate capability。

Preview、Export、CI对同一graph program、input、fixture、scheduler产生相同semantic trace/result；visual/UI差异由Scenario
assertion和Evidence负责。

## Diagnostics

现有`NGR-20xx/30xx/40xx/50xx`扩展为：

- typed port/edge/type/cycle planning；
- capability/runtime-zone/effect mismatch；
- async/retry/cancel/late completion；
- subgraph/reference/recursion；
- CodeSlot resolution/output；
- breakpoint/debug lease/value budget与replay divergence。

诊断必须定位document/node/port/CodeArtifact/Scenario step，meta不保存runtime value或Secret。

## 拒绝的方案

### 通用 JavaScript node

拒绝。任意代码通过typed CodeSlot进入，不能绕过Code Authoring/permission/budget。

### 每个node调用Remote API

拒绝。它破坏低延迟、确定性、取消和Export parity。

### graph output直接变成Workspace patch

拒绝。runtime patch不具备authoring intent、reverse op和revision preflight。

### debugger读取React Flow内部状态

拒绝。debugger消费domain execution snapshot/trace，React Flow只是投影。

## 后果

- NodeGraph wire/current model、validator、semantic provider和Compiler需要一次受控演进。
- Runtime Core增加可替换debug control/lease/value projection，不拥有graph document。
- Behavior Scenario可引用graph/step/checkpoint并接收normalized observation。
- Web NodeGraph editor增加typed ports、debugger和impact/Issues导航，不保存执行真相。

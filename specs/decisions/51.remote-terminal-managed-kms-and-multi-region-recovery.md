# ADR 51: Remote Terminal managed KMS and multi-Region recovery

- Status: Accepted
- Implementation: Implemented first vertical
- Date: 2026-07-19
- Owners: Remote Runner Control Plane, `@prodivix/runtime-remote`,
  `@prodivix/runtime-remote-postgres`

## Context

ADR 50 已让 Remote Terminal checkpoint 通过 opaque PostgreSQL row、revision CAS 与独立 Control Plane 副本继续运行，
但 PRT1 envelope 仍直接使用进程配置中的 32-byte AES master key。该模式适合本地开发和迁移，不是生产 managed-key
边界：数据库备份与 Control Plane 虽然分离，部署 Secret manager 仍需把长期解密 key material 直接投影给每个副本。

Environment Secret 的 AWS KMS adapter 不能直接复用为 Terminal cipher。两者 AAD、保存 schema、生命周期和故障语义不同；
Terminal 每个 stdin/output mutation 都可能形成新的短期 revision，并且 KMS 短暂故障不能被误判为密文损坏或触发 authority
row 删除。

跨 Region 时也不能把任意 KMS ARN 当作等价 key。单 Region key 的 ciphertext 只能由原 key 解密；只有同一 AWS
multi-Region key 的 primary/replica 共享 key id 与 key material。数据库复制与 KMS cryptographic portability 是两个独立
边界，本 ADR 只负责后者和复制后 exact authority row 的 broker continuation。

## Decision

### PRT2 envelope-key cipher

生产 Control Plane 可显式选择 `aws-kms` provider。每个 Terminal state revision：

1. 在 Control Plane 内生成新的随机 256-bit data key 与 96-bit AES-GCM nonce；
2. 以 data key 本地加密 bounded checkpoint；
3. 只把 32-byte data key 交给 managed KMS `Encrypt`；
4. 将 provider、logical key id、bounded wrapper metadata/ciphertext、state nonce/tag/ciphertext 编码为 strict `PRT2`；
5. 在 KMS 和 AES operation 完成后清零可变 data-key byte buffer。

PRT2 state AAD 绑定 execution id、terminal session id、revision、expiry、provider、logical key id，以及 KMS wrapper 的
SHA-256 identity。改变 row identity/expiry、替换 wrapped key/metadata、移动 ciphertext 或降级 envelope 都会 fail closed。
PRT2 最大 wrapped key 为 3 KiB，使最坏 envelope 仍处于 Runtime Remote `plaintext + 4 KiB` 与 PostgreSQL row hard cap 内。

### AWS KMS adapter

AWS first vertical 使用官方 JavaScript v3 SDK 和 standard credential chain。配置只接受：

- canonical region；
- 1-8 个 logical key id 到 exact immutable symmetric KMS key ARN 的映射，不接受 alias；
- exact active logical key id；
- 1-30000 ms operation timeout。

KMS encryption context 只有固定 purpose 与完整 Terminal data-key AAD 的 SHA-256；execution/session 等原始 identity 不进入
CloudTrail-visible context。Encrypt/Decrypt 请求显式使用 `SYMMETRIC_DEFAULT` 与 exact local ARN，响应必须返回同一 ARN、
algorithm、bounded ciphertext 和 exact 32-byte plaintext data key。wrapper metadata 在 KMS 调用前关联 logical key、stable key
identity、wrapped bytes 与 AAD，明显 drift 不消耗远端 decrypt quota。

AWS KMS 对 symmetric key 使用 encryption context 时要求 decrypt 提交 case-sensitive exact match；SDK `EncryptCommand` 只
处理小型 plaintext，因此 Terminal 只传 data key，不传 checkpoint。参考
[AWS KMS EncryptCommand](https://docs.aws.amazon.com/goto/SdkForJavaScriptV3/kms-2014-11-01/Encrypt)。

### Rotation and PRT1 migration

- static compatibility mode 继续读取/写入 PRT1；生产必须显式选择 `aws-kms` 才写 PRT2；
- managed rolling rotation 同时配置 old/new exact ARN，切换 active logical id；新 revision 自动用 active key 包装新的 data key；
- managed mode 可暂时保留 `REMOTE_TERMINAL_STATE_KEYS_JSON` 作为 decrypt-only PRT1 source；旧 row 首次成功 mutation 后由
  同一 CAS 写成 PRT2；
- 移除旧 managed/static key 后，仍引用它的 row fail closed，不尝试 key guessing 或 last-write-wins migration。

### Multi-Region recovery

每个 Region 部署相同 logical key id，但配置该 Region 的 exact local MRK ARN。PRT2 对 MRK 保存稳定
`partition/account/mrk-* resource id` identity，而 Decrypt request/response仍要求 exact local ARN。这样复制到另一个 Region 的
同一 opaque authority row 可以由 related replica 本地解密，随后由当地 broker 继续 token、mailbox、cursor 和 CAS；unrelated
MRK、不同 account/partition、single-Region ARN region drift 都被拒绝。

AWS 明确规定 related multi-Region primary/replica 具有相同 key id，可在一个 Region 加密、另一个 Region 用 related replica
解密；multi-Region ciphertext 的可解密地理范围也必须作为安全决策单独批准。参考
[How multi-Region keys work](https://docs.aws.amazon.com/kms/latest/developerguide/mrk-how-it-works.html) 与
[Security considerations for multi-Region keys](https://docs.aws.amazon.com/kms/latest/developerguide/mrk-when-to-use.html)。

本能力不自行复制 PostgreSQL。regional database replication/failover 必须保持 exact
execution/session/revision/expiry/sealed-state row，且 execution repository/worker lease 也必须在目标 Region 提供同一 current
authority，broker 才会继续；否则仍按 ADR 50 generation fence 关闭。

### Dependency failure semantics

`RemoteExecutionTerminalStateCipherUnavailableError` 是唯一 retryable cipher dependency failure。KMS timeout、throttling、5xx、
SDK retry exhaustion 等进入固定 `unavailable`；broker 不执行 CAS、不改变 token/mailbox/cursor，也不让 sweeper 删除 row。恢复后
相同 client sequence 可以安全重试一次。

invalid envelope、metadata/AAD drift、retired/unrelated key、KMS permission/identity response drift继续是 authenticated-state
failure，不标记 retryable。错误输出不反射 ARN、AWS request id、ciphertext 或 Secret material。

## Verification

- PRT2 independent cipher replicas、AAD/tamper、managed rotation/retirement 与 explicit PRT1 migration；
- AWS hashed encryption context、exact ARN/algorithm/response、pre-KMS metadata hard cut、timeout classification；
- related MRK primary/replica data-key unwrap 与 unrelated MRK rejection；
- 两个 regional broker composition 通过同一 opaque row 继续 stdin、worker mailbox、output cursor、token rotation 与 PRT2 CAS；
- runtime broker 在 cipher open/seal outage 前后保持 exact revision，恢复后不重复 stdin command；
- GitHub `G2 Managed KMS` workflow 同时运行本地 contract 和可选 live old/active rotation、MRK replica Gate。

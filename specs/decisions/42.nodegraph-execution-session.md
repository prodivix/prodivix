## 背景

NodeGraph 已有无 DOM deterministic executor，但 Blueprint trigger 曾通过 `@prodivix/runtime-browser` 的专用函数直接返回领域结果。该路径绕过 ExecutionJob、Session、取消、timeout、统一日志与 SourceTrace，也让浏览器 adapter 错误拥有 NodeGraph 执行语义。

NodeGraph 编辑器本身也没有正式 Run/Stop 产品入口，因此无法从作者表面验证当前 Canonical Workspace revision。

## 决策

`@prodivix/nodegraph` 拥有 same-context `ExecutionProvider`。它在领域 executor 之上完成 canonical Job 映射，不依赖 Window、DOM、React Flow 或 `@prodivix/runtime-browser`。

```mermaid
flowchart LR
    Workspace[Canonical Workspace revision] --> Request[NodeGraph ExecutionRequest]
    Request --> Provider[@prodivix/nodegraph provider]
    Provider --> Executor[Deterministic NodeGraph executor]
    Executor --> Trace[Domain trace + state patch]
    Trace --> Job[ExecutionJob events/result]
    Job --> Session[Execution Session coordinator]
    Session --> Console[Shared Execution Center]
    Session --> Blueprint[Blueprint runtime state patch]
```

### Revision-bound 输入

Web composition root 在运行前等待当前画布 Command 链完成，并重新读取 Canonical Workspace `pir-graph` document。每次 request 携带完整 Workspace snapshot identity、partition revisions、document-qualified diagnostic target、source owner、trigger 与 event key。

Provider 通过 host resolver 按 `requestId` 读取本次注册的 immutable document。Graph 内容不塞入 metadata，也不由 React Flow state 直接执行。

### Job 映射

Provider 支持 `preview/test` profile、`client/test` runtime zone 与 `nodegraph` invocation，并声明 cancellation、timeout、streaming logs、diagnostics 和 source trace capability。

- graph/node trace 实时映射为 ordered Job trace；
- `log` trace 同时映射为 console log；
- node event 使用 `nodegraph-node` SourceTrace，graph event 使用 document target；
- completed result 以 transport-safe output 返回 steps、state patch 与可选领域 output；
- no-entry、unsupported-node、missing-target 与 max-steps fail closed；未注册 node 使用 `NGR-1001`；
- 取消通过 cooperative signal 进入 executor；timeout scheduler 是可注入 host port，核心包不依赖 DOM/Node timer 类型。

Session identity 为 `workspace:{workspaceId}:nodegraph:{documentId}`。新 revision Job 替换当前 Job 并保留有界事件；重复运行先请求取消旧 Job，不留下编辑器私有进程或日志 store。

### 产品入口

NodeGraph editor 提供 Run/Stop，并在底部消费共享 Execution Center。Blueprint `run-nodegraph` trigger 使用同一 client、provider、session 与 result decoder；state patch 只应用到临时 Blueprint runtime state，不写回 Workspace。

旧 `executeNodeGraphAction` browser 直调协议与 `@prodivix/runtime-browser -> @prodivix/nodegraph` 依赖已删除，不保留兼容层。

## 当前能力边界

默认 registry 确定性支持 `start/process/switch/log/end`，并保证新建 starter graph 可完整运行。其他节点只有在 registry 注册正式 executor 后才能执行，Web 不按 palette label 猜测行为。

G3 继续完成 typed data flow、async/error/cancel、state/transaction、subgraph、breakpoint、step/continue、CodeSlot executor、Behavior Scenario 与 VerificationEvidence。需要 network、Secret、server 或 edge 的节点必须选择具有相应 capability 和 runtime zone 的 provider。

Animation 已在 `43.animation-runtime-and-execution-session.md` 建立可替换 scheduler、generation-fenced effect host、完整单 timeline lifecycle、SourceTrace 与 target capability contract，并接入同一 Session/Console。G3 继续完成跨 timeline/route composition、reduced-motion、CodeSlot/shader 与 VerificationEvidence。

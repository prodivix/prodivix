# ADR 55: Data Stream Recovery、Credential Renewal 与 Incremental Collection

- Status: Accepted
- Implementation: Same-execution recovery product vertical implemented
- Date: 2026-07-20
- Owners: `@prodivix/data`、GraphQL/AsyncAPI adapter、`@prodivix/runtime-core`、Backend Remote Execution、`apps/web`
- Revises: ADR 49 中 reconnect/resume、Secret stream 与 incremental collection 的 fail-closed 边界

## Context

ADR 49 建立了 server/edge GraphQL/AsyncAPI pull-driven stream、预算、Remote authority 与 SourceTrace first
vertical，但有意不允许断线恢复、Secret 长连接和增量集合。真实 Preview stream 仍需要在不创建新 execution、
不自动重放 mutation、不向 capability iframe 暴露 credential/checkpoint 的前提下恢复瞬时 SSE 断线，并把事件
确定性投影为有界 collection snapshot。

仅把 SSE `Last-Event-ID` 或 bearer 放进现有 iframe wire 会破坏 ADR 49 的 value-only 边界；只编码未认证的
cursor token 又允许客户端修改 duration/byte/cursor budget。因此 recovery 必须是 Backend 与 product client
之间的私有协议，iframe 仍只看到原有 open/pull/event/complete/error v1 contract。

## Decision

### Canonical stream policy

`DataOperationPolicies.stream` 只允许 subscription，并固定以下显式 current contract：

- `reconnect.resume` 只能是 `sse-last-event-id`；attempt、fixed/exponential backoff 与 delay 都有全局上限；
- Secret-authenticated stream 必须声明 `credentialRenewal: per-connection`，无 Secret 时不得伪声明 renewal；
- 可选 `collection` 只能是 `keyed-event-v1`，声明 RFC 6901 `entityIdPath` 与最多 10,000 items；
- subscription 继续禁止 finite cache/retry/idempotency/pagination/optimistic policy。

Workspace TypeScript/Go validator、Compiler target Gate、Remote snapshot 与 protocol adapter 消费同一 policy；
literal/public environment authorization 不能冒充 Secret renewal。

### Same-execution SSE recovery

- Backend 只为声明 recovery policy 的 SSE stream 生成 checkpoint；NDJSON、无 SSE id 或 malformed id fail closed。
- checkpoint 绑定 execution/document/operation/invocation/sequence、当前 cursor、upstream SSE id、原始 openedAt 与累计
  bytes。Backend 进程随机生成的 256-bit key以 HMAC-SHA-256 认证 payload；客户端不能修改预算或 cursor。
- product client 只在收到 retryable protocol/transport failure、拥有上一事件 checkpoint 且未超过 attempt budget 时重连，
  并以 `Last-Event-ID` 恢复同一 invocation。event cursor 必须继续精确递增；总 record/byte/duration budget不重置。
- recovery 不创建新 ExecutionRequest/Job，不跨 execution replay，不恢复 mutation，也不把 event history 写入 Workspace。
- 当前 HMAC key 是 Backend process-local。跨副本恢复安全地 fail closed；共享 managed key/MRK 与真实 multi-region
  evidence 按用户安排留到 AWS/KMS 阶段，不能把本地通过描述为跨实例可用。

### Credential renewal and leak boundary

- 每次 initial/reconnect connection 都重新获得 product auth token，并在 Backend 内签发新的 exact execution/environment
  grant；Secret 只在 `UseSecret` callback 内注入 handshake header，open 后立即删除 header并 revoke grant。
- Backend 对本次连接的 raw credential 与 Bearer form 保存 SHA-256 fingerprint，只拒绝 decoded event 中的 exact scalar
  echo；任何 Secret value、upstream SSE id、resume token均不进入 sanitized Network、Session trace或 iframe event。
- Backend NDJSON 的 `reconnect`/`resume` 字段只由 product client strict decoder 消费并剥离。Runtime Core v1 decoder继续
  拒绝这些私有字段；iframe wire无版本变化。

### Network correlation

每次 connection 都产生独立 sanitized Network request identity `invocation:stream:cursor`，共享 exact Data operation
correlation、SourceTrace、active Job、Session 与 generation fence。initial trace仍随 iframe open 发布；reconnect trace只进入
Execution Center Session，不向 application frame暴露 checkpoint或 credential。trace publication conflict、stale Job 或
consumer exception终止 stream。

### Incremental collection

声明 `keyed-event-v1` 后，事件值只接受三个 exact envelope：

```text
{ action: "replace", items: [...] }
{ action: "upsert", entity: {...} }
{ action: "delete", id: string | safe-integer }
```

Data kernel与 generated standalone runtime执行同一 exact-cursor、identity uniqueness、capacity和 immutable snapshot
规则。每个 event先通过 operation output schema，再原子 apply；失败不会产生部分 snapshot。snapshot 是 execution-local
可丢弃状态，包含 cursor、appliedEvents 与 items，不写 Canonical Workspace、Outbox 或 durable event history。

## Verification

`pnpm run verify:g2:data-stream-debugger` 覆盖：

1. strict stream policy codec、incremental replace/upsert/delete、cursor/identity/capacity hard cut；
2. GraphQL/AsyncAPI Secret connection只在 environment lease内打开；
3. Backend HMAC checkpoint、Last-Event-ID、两次独立 Secret grant/use/revoke、credential echo与 tamper denial；
4. product client动态 bearer renewal、透明 resume、私有字段剥离、共享总预算；
5. exact active Job reconnect Network observation与 unsubscribe/termination fence；
6. generated runtime schema-first incremental collection，以及 Runtime Core对 `reconnect`/`resume` iframe字段的拒绝；
7. React/Vue target Golden matrix、Web typecheck与 Backend Workspace/Remote Execution tests。

2026-07-20 本地 Gate：Runtime Core 21 files / 91 tests、Data 14 / 58、GraphQL 2 / 9、AsyncAPI 2 / 9、
Compiler 2 / 23、Golden 2 / 20、Web 7 / 39，Backend Workspace与Remote Execution Go packages均通过。

## Non-goals and remaining work

- WebSocket/GraphQL WS、Kafka、MQTT、durable event log、cross-execution replay与跨副本 checkpoint availability不在本纵切；
- AWS KMS/MRK、OIDC Role、GitHub Environment/Secrets与真实 multi-region evidence按用户要求暂缓；
- exact scalar credential echo guard不替代后续完整跨表面 Secret substring/canary matrix；
- incremental snapshot不自动改写 PIR durable binding，也不提前定义 G3 Behavior/Verification Evidence。

## Consequences

- public 与 Secret SSE stream均可在同一 execution中有界恢复，同时保持 value-only iframe contract不变；
- 每次重连拥有新的 credential/grant和独立 Network evidence，而不会复活 terminal Job；
- Collection消费者得到确定性的 immutable runtime projection，但 Canonical Workspace仍是唯一作者态真相；
- 跨 Backend 副本可用性仍依赖后续 managed shared-key阶段，当前只承诺安全 fail closed。
